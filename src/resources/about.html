Автоматизированная система генерации лабиринта и нахождения выхода из него

Курсовой проект по дисциплине

«Современные методы разработки и проектирования сложных программных комплексов» 

Исходные данные к проекту:

Характеристика объекта автоматизации:

  • объект автоматизации: двухмерный ортогональный лабиринт;
  • виды автоматизируемой деятельности:
  • процесс настройки параметров лабиринта;
  • процесс генерирования лабиринта;
  • процесс поиска выхода из лабиринта;
  • процесс визуализации работы с лабиринтом;
  • минимальный размер лабиринта по горизонтали – 7 клеток;
  • максимальный размер лабиринта по горизонтали – 37 клеток;
  • минимальный размер лабиринта по вертикали – 7 клеток;
  • максимальный размер лабиринта по вертикали – 37 клеток;
  • количество входов – 1;
  • количество выходов – 1;
  • количество тем оформления лабиринта – 4;
  • количество способов расстановки входа и выхода –2;
  • количество алгоритмов генерации лабиринта – 2;
  • количество алгоритмов поиска выхода из лабиринта –2;
  • количество способов визуализации прохождения лабиринта – 2;
  • количество режимов динамической визуализации – 3.

Требования к информационному обеспечению:

информационное обеспечение разрабатывается на основе следующего источника:

  • Алгоритмы создания лабиринта [Электронный ресурс]. URL: https://en.wikipedia.org/wiki/Maze_generation_algorithm (дата обращения: 29.09.2020);
  • Алгоритмы поиска выхода из лабиринта [Электронный ресурс]. URL: https://en.wikipedia.org/wiki/Maze_solving_algorithm (дата обращения: 29.09.2020);

Требования к структуре лабиринта:

  • единицей декомпозиции схемы лабиринта является клетка;
  • толщина стены и толщина прохода одинаковые и составляют 1 клетку;
  • вход и выход должны находиться на периметре лабиринта, не совпадать и не находится в углах;
  • размер лабиринта должен быть нечетным;
  • лабиринты хранятся в файлах, структура файла определяется в процессе проектирования.

Требования к техническому обеспечению:

  • тип ЭВМ – IBM PC совместимый;
  • монитор с разрешающей способностью не ниже 800 х 600;
  • манипулятор – мышь;
  • технические характеристики определяются в процессе выполнения проекта.
  • Требования к программному обеспечению:
  • тип операционной системы ‑ Windows 7 и выше;
  • язык программирования – Java 8+;
  • среда программирования – IntelliJ IDEA Community Edition 2019.3.4;
  • среда проектирования – StarUML 4.0.0, yEd 3.17.1, Balsamiq Mockups 3.99
  • Общие требования к проектируемой системе:

Функции, реализуемые системой:

  • общесистемные функции:
    • автоматическая генерация лабиринта по заданным параметрам;
    • визуализация процесса прохождения лабиринта;
    • выдача справочной информации о системе;
    • автоматическая расстановка входа и выхода;
    • поиск выхода из лабиринта в соответствии с выбранным алгоритмом;
  • функции пользователя:
    • загрузка лабиринта из файла;
    • сохранение лабиринта в файл заданной структуры;
    • настройка параметров лабиринта при создании:
  • задание размера сетки по горизонтали;
  • задание размера сетки по вертикали;
  • выбор одной из 4 цветовых тем оформления лабиринта;
  • выбор режима расстановки входа и выхода (автоматический/ручной);
  • выбор режима поиска выхода из лабиринта (автоматический/ручной);
  • выбор режима визуализации (статическая/динамическая);
  • настройка параметров динамической визуализации;
    • ручное прохождение лабиринта;
    • сохранение лабиринта в файл.
    • загрузка лабиринта из файла.

Технические требования к системе:

  • режим работы ‑ диалоговый;
  • время автоматической генерации лабиринта – не более 15 с;
  • система должна удовлетворять санитарным правилам и нормам
    СанПин 2.2.2./2.4.2198-07;
  • условия работы средств вычислительной техники (содержание вредных веществ, пыли и подвижность воздуха) должны соответствовать ГОСТ 12.1.005, 12.01.007;
  • температура окружающего воздуха – 15-35°С;
  • влажность воздуха – 45-75%.
